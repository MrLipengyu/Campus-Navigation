题目 05 校园探索与智能导航模拟系统 Qt / UE5 ★★★☆☆
5.1 题目背景
新生入学时常常对校园地理环境不熟悉，传统的纸质地图和静态电子地图难以提
供沉浸式的浏览体验。本课题要求开发一款校园探索与导航模拟系统，支持地图浏
览、楼宇信息查询、最短路径规划与角色漫游。本题目特别开放两条技术路线，小组
可根据自身兴趣与特长任选其一：
● 【Qt 6 路线】使用 Graphics View Framework 实现 2.5D 俯视/斜视视角的校
园地图，重点在于图形界面、地图数据建模与路径算法的实现。
● 【UE5 路线】使用 UE5 搭建 3D 校园场景，玩家以第一人称或第三人称视角自
由漫游真实比例的校园，重点在于 3D 场景搭建、角色控制与导航系统的集
成。
无论选择哪条路线，核心要求一致：路网建模、路径规划、信息查询与漫游交互
必须完整实现。
5.2 基础功能（必做）
● 校园地图：构建包含至少 15 个建筑物（教学楼、图书馆、宿舍、食堂、体育
馆等）和连接道路的校园地图；Qt 路线为 2.5D 俯视图，UE5 路线为 3D 场
景。
● 路网建模：使用图结构（邻接表或邻接矩阵）建模道路网络，节点为建筑物或
路径关键点，边带权重（距离/步行时间）。
● 角色控制：用户可控制一个角色在校园中漫游——Qt 路线为 2.5D 精灵图，使
用 WASD 移动并有 4 方向行走动画；UE5 路线为 3D 角色（可使用
Mannequin），支持第一/第三人称切换。
● 最短路径规划：用户可选择起点与终点（点击地图建筑物或在搜索框输入名
称），系统使用 Dijkstra 或 A* 算法计算最短路径，并在地图上以高亮线条
/ 3D 引导箭头显示。
● 建筑信息查询：点击 / 走近建筑物时弹出信息面板，显示名称、简介、开放时
间、楼层平面图等；使用 SQLite（Qt 路线 QSqlDatabase / UE5 路线第三方
库或 UDataTable）存储信息，支持后台管理员模式增删改查。

● 自动导航：选择目的地后，角色可自动沿规划路径行走至目标位置（按一定速
度自动移动）。
5.3 扩展功能（选做）
● 昼夜系统：根据系统时间或手动调节切换日间/夜间，整体光照与配色改变（Qt
路线为色调切换，UE5 路线为天空与方向光动态调整）。
● 天气模拟：实现下雨、下雪等天气特效（Qt 路线使用 QPainter 粒子，UE5 路
线使用 Niagara）。
● NPC 系统：地图上随机生成会移动的 NPC（学生/教师），玩家可与之对话获取
趣味提示。
● 迷你地图：在主视图角落显示全局迷你地图与当前位置（UE5 路线需额外用
RenderTarget 实现）。
● 多终点路径：用户可设置多个目的地，系统使用 TSP 近似算法规划最短遍历顺
序。
● 室内楼层：选择某栋楼后可进入室内楼层视图，查看教室分布。
5.4 技术要点
【 Qt 路 线 】 QGraphicsScene/QGraphicsView/QGraphicsItem 体 系 ； 自 定 义
QGraphicsItem 实现动态精灵；图算法（邻接表+优先队列）；QPropertyAnimation；
信号与槽机制；QSqlDatabase 数据持久化。
【UE5 路线】Character / Pawn / Controller 体系；Enhanced Input 输入系
统；UE5 内置 NavMesh 寻路或自定义 Dijkstra/A* 算法；UMG 信息面板与 UI 交
互；UDataTable 配置建筑信息；Lumen 光照与日夜循环。
两条路线难度相当，最终评分标准一致。

2.1 技术要求
● 开发语言：以 C++（C++17 或以上标准）为主，禁止使用其他语言完成核心业
务逻辑。
● 可视化框架：选用 Qt 6（推荐 Qt 6.5 LTS 及以上）或 Unreal Engine 5（推
荐 5.3 及以上）。题目中已明确指定推荐框架。
● 构建系统：必须使用 CMake、qmake 或 Unreal Build Tool 进行项目构建，禁
止手工管理编译命令。
● 版本控制：必须使用 Git，并将代码托管至 Gitee、GitHub 或学校内部
GitLab，组内成员均需有提交记录。
● 编码规范：参考 Google C++ Style Guide 或 Qt Coding Style，要求注释覆
盖率不低于 20%，关键算法必须有详细注释。
● 第三方库：可使用 STL、Qt 自带模块、Unreal 自带模块；如需引入其他第三
方库（如 OpenCV、Eigen、nlohmann/json、Box2D），须在报告中说明。
