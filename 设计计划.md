# 校园探索与智能导航模拟系统十五天开发计划书

## 一、项目定位

本项目拟开发一个基于 Qt 6 的校园探索与智能导航模拟系统。系统采用 2.5D 俯视校园地图形式，支持校园建筑展示、道路网络建模、角色漫游、建筑信息查询、最短路径规划、自动导航、迷你地图、昼夜切换等功能。

项目开发重点不是单纯绘制地图，而是建立清晰的程序结构：地图数据层负责保存建筑、道路、节点；图算法层负责路径计算；可视化层负责地图与角色显示；控制层负责把用户操作、路径规划、角色移动连接起来；数据库层负责建筑信息管理。

## 二、总体开发思路

项目从第一天开始采用最终版架构，不再先做临时地图再重构。核心设计原则如下：

1. 建筑物 Building 与路径节点 Node 分离。建筑中心点只用于显示，真正寻路使用道路节点或建筑入口节点。
2. 路网采用图结构建模，节点表示道路关键点、路口、建筑入口，边表示可行走道路，边权为距离或步行时间。
3. UI 层只负责界面显示，不直接操作算法。
4. Graph 与 Dijkstra 模块保持独立，方便测试和答辩讲解。
5. NavigationController 作为控制层，统一处理“选择起点终点—计算路径—高亮路线—角色自动移动”的流程。
6. 数据库模块后期接入 SQLite，用于保存建筑名称、简介、开放时间、楼层信息等。

建议项目目录如下：

CampusNavigator/
├── main.cpp
├── CMakeLists.txt
├── ui/：主窗口与界面
├── map/：建筑、节点、道路、校园地图数据
├── graph/：图结构与 Dijkstra 算法
├── graphics/：地图图元、角色图元、路径高亮
├── controller/：导航流程控制
├── database/：SQLite 数据管理
└── resources/：图标、地图、图片、样式文件

### Day 4：Dijkstra 最短路径算法实现

**主要目标：完成路径规划核心算法。**

需要完成的代码模块：

* graph/Dijkstra.h、Dijkstra.cpp

具体任务：

1. 使用优先队列实现 Dijkstra。
2. 输入起点节点 ID 和终点节点 ID。
3. 输出节点路径序列，例如：1 → 5 → 8 → 12。
4. 保存每个节点的最短距离。
5. 保存前驱节点，用于还原完整路径。
6. 处理异常情况：

   * 起点不存在
   * 终点不存在
   * 起点和终点不连通
   * 起点等于终点
7. 用 3—5 组测试数据验证算法正确性。
8. 准备答辩时能讲清楚：

   * 为什么使用 Dijkstra
   * 邻接表如何存储图
   * 优先队列如何优化搜索过程
   * 路径如何回溯生成

当天成果：

* 能在控制台或调试窗口输出最短路径。
* 算法模块稳定。
* 项目已有核心技术亮点。

## 五、答辩展示建议

答辩时建议按以下顺序演示：

1. 打开程序，展示校园整体地图。
2. 放大缩小、拖动画布，说明使用 Qt Graphics View 实现。
3. 点击图书馆或教学楼，展示建筑信息面板。
4. 设置起点和终点，点击规划路径。
5. 讲解路径规划基于图结构和 Dijkstra 算法。
6. 点击开始导航，展示角色自动行走。
7. 展示搜索建筑功能。
8. 展示迷你地图和昼夜切换。
9. 展示多终点路径规划。
10. 进入管理员模式，展示建筑信息增删改查。

## 六、风险控制

1. 不要前期过度追求地图美术，先保证数据结构和路径算法正确。
2. 不要把建筑中心直接作为寻路节点，否则后期自动导航会不自然。
3. Dijkstra 算法必须独立测试，不能只依赖界面测试。
4. 数据库功能放在 Day 11，不要太早接入，避免拖慢核心功能。
5. Day 14 之后不再新增大功能，只做修复和优化。
6. 答辩前必须准备一条稳定演示路线，例如“校门 → 图书馆 → 食堂 → 宿舍”。
